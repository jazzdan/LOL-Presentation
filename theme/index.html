LOL Object Language

Also known as...

LOL

Authors

Monday, May 7, 2011

Summary

LOL (pronounced “lawl”) is an object-oriented, dynamically typed programming language with an emphasis on asynchronous event-driven programming achieved through an implementation of the actor model.

Motivation

If you want concurrency and have only objects, the best way is to commit suicide.
Prof. Christian Schulte on OO Concurrency

Code Examples

Defining Actors


lol Person:
	this.name
	
	secret
	
	this.__constructor name, secret:
		this.name = name
		this.secret = secret
	end
	
	this.get_secret:
		return name + “ “ + secret
	end

	this.receive message, fromPid:
		echo message
		response = message + “... but it actually sucks.”
		this.send(fromPid, response)
	end

end

Language Evaluation Criteria

Overall Simplicity

  • It's the simplest.
  • Very few build in methods (except for concurrency because that's hard.
  • This makes it a great learner's language!

Language Evaluation Criteria

Data Types and Structures

  • Only the data types you would expect.
  • No built-in advanced data structures.
  • No built-in classes (again, except for concurrency).
